Overview

Draw2Code is a paper-based computational kit for early childhood to create interactive AR animation. With pen, paper and a mobile device, young children can learn computational thinking by programming and playing in physical world and to produce and interact with virtual content.


Background

Computational thinking is nowadays considered an essential skill in the K-12 educational curriculum. While many computational kits and platforms have been developed in the past decades, most of the tangible kits that are designed for early childhood are either expensive to be deployed at home and in classroom environments or only provide pre-defined challenges with limited creative content creation opportunities. We investigated low-cost and expressive tangible interfaces that foster computational literacy.


Design Goals

The intended audience of Draw2Code is young children of pre-school and early elementary school. In addition to being age-appropriate, intuitive, and attractive, Draw2Code has three design goals:

  1. Support hands-on experiences by offering natural interaction and supporting physical engagement. We achieve this by using coding by drawing and interacting through gestures.
  2. Be low-cost, using affordable materials. We address this by using pen & paper and widely available personal mobile devices such as cell phones and tablets.
  3. Provide openness in which children can express personally meaningful ideas. We tackle this goal by using a freeform drawing that is familiar and expressive.


Interface Design

The Draw2Code kit consists of two major components: paper coding blocks and a mobile application. Children can author their own animations by manipulating the paper coding blocks and execute the code with a mobile device. We designed elements of the interface to make both of tangible and virtual interfaces appropriate for young children. For example, no text is used, buttons are big enough, and icons are straightforward to understand. We also paid attention to involving both boys and girls, such as using colors to which both genders can feel a sense of belonging.




The Sprite block has an empty canvas area where children can draw their own character or place a small craft on it to use it as a character. The Event block is for triggering the virtual character to animate. The current version can use hand gestures such as rock paper scissors and graphical play button as event blocks. Each event block represents a particular command to trigger the program and has a unique visual depending on its type. The Action block has a grid where children can fill color in it with a different size and location that guide the virtual characterâ€™s appearance in each frame. A sequence of action blocks creates an animation.



The child can use the scan mode in the mobile application to load the sequence of the blocks as an animation and see the current state of the animation in the run mode. Children make their paper drawing alive as an animated virtual sprite and control it using hand gestures.



Through out the process, they go through five steps: design, code, scan, play, and test. It exposes children to basic programming concepts through playful and tangible interaction.


Implementation

We fabricated Draw2Code paper coding blocks by printing them in colored paper and cutting out the paper.



Draw2Code software is a web-based application written in Javascript. We used the p5.js library for building user interface components. We leveraged Teachable Machine to create image classification models to detect different coding blocks and hand gestures from images. We made image processing algorithms to parse the block images to data and control variables using computer vision techniques. We used a camera embedded in a mobile device to capture the block images.



Evaluation

We conducted user study with 13 pairs of child-parent participants to evaluate the usability of Draw2Code interface. Children's age was between 5 and 13. Overall, children reported they enjoyed working with Draw2Code and all of them successfully completed tasks. Children played with Draw2Code for approximately 20 minutes up to 45 minutes, excluding tutorials, showing high engagement. Most children mentioned they particularly enjoyed the moment when their drawing came to life on the AR screen.
